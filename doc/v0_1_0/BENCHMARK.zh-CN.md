# 性能测试

**测试环境：** Windows 10 x64 | Julia 1.12.6 | CPU: i7-12700H | OpenSSL 3.x

所有测试采用中位数延迟（50-5000 次迭代，5 次预热）。
OpenSSL 对比基于 EVP API（通过 `libcrypto` 调用）。
流式测试基于 64 字节分块 `sm4_stream_update!` / `update!` 模式，模拟真实流处理场景。

---

## SM9 标识密码

| 操作            | 延迟    |
|-----------------|---------|
| 主密钥生成       | 4.04 ms |
| 用户密钥提取     | 2.32 ms |
| G1 哈希到点      | 3.79 ms |
| G1 标量乘法      | 3.70 ms |
| 参数验证         | 6.9 us  |

SM9 基于 256 位 BN 曲线，包含完整域扩展塔（Fq -> Fq2 -> Fq12）和 Ate 双线性配对，全部纯 Julia 实现。OpenSSL 无 SM9 支持可供对比。

## 总结

| 算法      | 类型          | 纯 Julia 吞吐量 | vs OpenSSL   |
|-----------|---------------|-----------------|--------------|
| SM4 ECB   | 分组密码      | 115-127 MB/s     | 1.5x 慢      |
| SM4 CBC   | 分组密码      | 111-125 MB/s     | 1.6x 慢      |
| SM4 CTR   | 流式分组密码  | 110-119 MB/s     | N/A          |
| SM4 CBC流式| 流式分组密码  | 114-124 MB/s     | N/A          |
| SM4 CFB流式| 流式分组密码  | 111-123 MB/s     | N/A          |
| SM4 OFB流式| 流式分组密码  | 110-120 MB/s     | N/A          |
| SM3       | 哈希          | 222 MB/s（流式） | N/A          |
| ZUC       | 流密码        | 201-236 MB/s     | N/A          |
| SM2 签名  | ECC           | 3.8 ms/次        | 6.4x 慢      |
| SM2 验签  | ECC           | 5.5 ms/次        | 18.4x 慢     |
| SM9       | 标识密码      | 2-4 ms/次        | N/A          |

- **SM4**：纯 Julia 同类最佳性能，仅比 C+汇编慢 1.4-1.6 倍。Sm4Stream 流式 API 覆盖全部 5 种模式（ECB/CBC/CFB/OFB/CTR），ECB/CBC 默认启用 PKCS7 填充；T-table 预计算带来约 1.5 倍提速
- **SM3**：分块流式哈希 **222 MB/s**。就地 CF! 和可复用 W 缓冲区消除逐块内存分配，性能优秀
- **ZUC**：4 字节批量密钥流提取实现峰值 **~236 MB/s**，相比逐字节处理提升约 4 倍，无中间密钥流缓冲区

# 一次性运行全部 standalone 测试：
julia --project=. demo/benchmark/run_benchmarks.jl

# 含 OpenSSL 对比（需 OpenSSL_jll）：
